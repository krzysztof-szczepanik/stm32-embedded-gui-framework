Detailed Description

Base screen structure used as the root widget of a view.

A screen acts as a container for widgets, manages focus navigation, and propagates events and layout updates through the widget tree.

void(* ui_screen::update_data) (ui_screen_t *screen)

Optional screen update callback.

This function is called automatically whenever the screen becomes active. It allows the screen to refresh its internal widget values based on the shared UI state stored in ui_screens_data.

Typical use cases:

  • synchronizing widget values with settings changed on another screen
  • restoring slider/toggle states
  • updating labels before the screen is drawn
  • reacting to global state changes (e.g. timer state)

Each screen may provide its own implementation of this function. If NULL, no update is performed.

Parameters
screenPointer to the screen instance being activated.
